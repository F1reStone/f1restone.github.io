本指南将带你完成运行 Astro Rocket 所需的全部步骤，包括本地运行、品牌化配置以及部署到互联网。无需任何 Astro 使用经验。

## 开始之前你需要准备的内容

* [Node.js](https://nodejs.org) 22.12.0 或更高版本
* 一个包管理器：`npm`、`pnpm` 或 `yarn`
* 一个用于部署的免费 [Vercel 账号](https://vercel.com)
* 一个代码编辑器 —— 推荐使用安装了 [Astro 扩展](https://marketplace.visualstudio.com/items?itemName=astro-build.astro-vscode) 的 [VS Code](https://code.visualstudio.com)

## 安装

Astro Rocket 是免费且开源的。从 GitHub 克隆仓库并安装依赖：

```bash
git clone https://github.com/hansmartensdev/astro-rocket my-site
cd my-site
pnpm install
pnpm dev
```

打开 `http://localhost:4321`，你就能看到完整网站已在本地运行。你所做的每一次修改都会即时更新，无需整页重新加载。

## 配置你的网站

网站的大部分配置都集中在一个文件中：`src/config/site.config.ts`。

```ts
const siteConfig: SiteConfig = {
  name: 'Your Site Name',
  description: 'A short description for search engines.',
  url: 'https://yoursite.com',
  author: 'Your Name',
  email: 'hello@yoursite.com',
```

从这里开始。`name` 字段会用于 Logo 徽标、页面标题、页脚版权信息以及结构化数据。`description` 会作为未单独提供描述页面的默认 Meta Description。

### 品牌配色

Astro Rocket 内置了 12 套可直接使用的颜色主题。你可以在 `src/layouts/BaseLayout.astro` 中通过设置 `data-theme` 属性来选择喜欢的主题：

```html
<html lang="en" class="scroll-smooth dark" data-theme="indigo">
```

可用值包括：`orange`、`amber`、`lime`、`emerald`、`teal`、`cyan`、`sky`、`blue`、`indigo`、`violet`、`purple`、`magenta`。主题 CSS 文件位于 `src/styles/themes/` 中 —— 如有需要，可以打开当前主题文件来微调 `--brand-*` 色阶。

`site.config.ts` 中的 `branding.colors` 部分用于控制两个与页面样式无关的浏览器参数：

```ts
branding: {
  colors: {
    themeColor: '#6366f1',       // 移动端 Chrome/Safari 的浏览器工具栏颜色
    backgroundColor: '#ffffff',  // PWA 启动画面的背景色
  },
},
```

将 `themeColor` 更新为与你品牌颜色匹配的十六进制值，这样移动端浏览器的界面会更加统一美观 —— 它不会影响页面中的实际颜色。

### 导航栏

打开 `src/config/nav.config.ts`。这是你修改网站导航时唯一需要调整的文件：

```ts
export const navItems: NavItem[] = [
  { label: 'Blog',     href: '/blog',     order: 1 },
  { label: 'Projects', href: '/projects', order: 2 },
  { label: 'About',    href: '/about',    order: 3 },
  { label: 'Contact',  href: '/contact',  order: 4 },
];
```

你可以添加、删除或重新排序项目。页头和页脚都会自动读取这份配置。

## 编写博客文章

在 `src/content/blog/en/` 中创建一个新的 `.mdx` 文件。使用以下 frontmatter 模板：

```mdx
---
title: "Your Post Title"
description: "A short summary under 200 characters."
publishedAt: 2026-03-15
author: "Your Name"
tags: ["tag-one", "tag-two"]
featured: false
locale: en
image: "../../../assets/blog/your-image.svg"
imageAlt: "Describe the image for screen readers."
---

Your post content starts here. Standard Markdown works — headings,
lists, links, bold, italic — plus any MDX components you need.
```

保存文件后，文章会立即出现在博客索引中。无需重新构建，也无需清除缓存。

### 封面图片

博客文章封面图片位于 `src/assets/blog/` 中。推荐使用 1200×630 像素的 SVG 格式 —— 这是标准的 Open Graph 图片尺寸。SVG 可以在保持极小文件体积的同时完美适配各种屏幕。

如果你更倾向于使用照片，PNG 和 JPG 也同样适用；Astro 的 `<Image>` 组件会自动完成优化。

## 配置联系表单

联系表单使用 [Resend](https://resend.com) 发送邮件。要启用它：

1. 创建一个免费的 Resend 账号并获取 API Key
2. 在 Resend 中验证你的发送域名
3. 在 `src/config/site.config.ts` 中设置你的 `email` —— 表单提交内容会发送到该地址
4. 将 API Key 添加到 `.env`：

```
RESEND_API_KEY=re_xxxxxxxxxxxx
```

你还可以选择设置 `RESEND_FROM_EMAIL`，并填写一个已验证的域名邮箱地址。若未设置，表单会使用 `site.config.ts` 中的 `email` 作为发件地址。

即使没有这些变量，表单也不会报错 —— 它只是会禁用提交功能。因此你可以先开发和部署网站的其余部分，再后续配置邮件功能。

## 部署到 Vercel

1. 在 GitHub（或 GitLab / Bitbucket）上创建一个新的仓库，并将本地项目推送上去
2. 打开 [vercel.com](https://vercel.com)，点击 **Add New Project**
3. 导入你的仓库 —— Vercel 会自动识别 Astro 项目
4. 在 **Settings → Environment Variables** 中添加环境变量
5. 点击 **Deploy**

至此，你的网站已经上线。之后每一次推送到 `main` 分支都会自动触发新的部署。每个分支也都会自动生成 Pull Request 预览。

将你的 `SITE_URL` 环境变量设置为生产环境域名：

```
SITE_URL=https://yoursite.com
```

这样可以确保 Canonical URL、站点地图以及结构化数据都使用正确的域名。

## 部署到 Netlify

Astro Rocket 同样开箱即用支持 [Netlify](https://netlify.com)。项目已经附带了配置正确构建参数与安全头的 `netlify.toml` 文件。

1. 在 GitHub（或 GitLab / Bitbucket）上创建一个新的仓库，并将本地项目推送上去
2. 打开 [netlify.com](https://netlify.com)，点击 **Add new site → Import an existing project**，然后连接你的仓库
3. 在 **Site configuration → Environment variables** 中添加你的环境变量，包括：

```
SITE_URL=https://yoursite.com
DEPLOY_TARGET=netlify
```

4. 点击 **Deploy site**

设置 `DEPLOY_TARGET=netlify` 会让构建过程使用 Netlify Adapter，而不是默认的 Vercel Adapter。除此之外，其余功能 —— 包括联系表单、API 路由以及图片优化 —— 在两个平台上的表现完全一致。

## 接下来该做什么

网站上线后，推荐按以下顺序继续完善：

1. **替换占位内容** —— 更新 Hero 文案、关于页面以及首页内容，使其反映你的真实项目与工作内容
2. **发布第一篇正式文章** —— 持续发布内容是提升搜索可见性最有价值的事情之一
3. **配置 Google Search Console** —— 在 `.env` 文件中添加 `GOOGLE_SITE_VERIFICATION`，并提交你的 Sitemap
4. **添加社交链接** —— `site.config.ts` 中的 `socialLinks` 数组会自动生成页脚社交图标
5. **配置统计分析** —— 添加 `PUBLIC_GA_MEASUREMENT_ID` 或 `PUBLIC_GTM_ID` 来启用内置分析集成
